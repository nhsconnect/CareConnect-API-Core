Return a single {{include.resource}} for the specified id{{include.content}}.

All requests MUST contain a valid ‘Authorization’ header and MUST contain an ‘Accept’ header. The `Accept` header indicates the format of the response the client is able to understand, this will be one of the following application/fhir+json or application/fhir+xml.

1.1. Response

A full set of response codes can be found here API Response Codes. FHIR Servers MUST support the following response codes:

200 successful operation
400 invalid parameter
401/4xx unauthorized request
403 insufficient scope
404 resource not found
410 resource deleted
